# Вариант 2.

Основная проблема каждого задания - подключена namespace std.
Использование using namespace std считается плохой практикой, так как
оно переносит все имена из стандартной библиотеки в глобальную область видимости.
Это вызывает конфликты имён, делает код менее читаемым, затрудняет отладку и может
привести к непредсказуемым ошибкам компиляции, особенно при обновлении стандартов C++
или подключении сторонних библиотек.

Вторая не менее важная проблема почти каждого задания - отсутствие разбиения программы
на отдельные методы. Разбиение программы на отдельные методы необходимо для читабельности и  масштабируемости, упрощения тестирования и отладки, последующего интегрирования данного метода
в другие программы, имеющие подобные задачи.

# Задание 1:
## Описание задачи
Археолог нашел N артефактов. Известны веса (cᵢ) и налоговое бремя (dᵢ) находок. 
Требуется выбрать такое подмножество находок, чтобы их суммарный вес превысил Z кг, 
а их общее налоговое бремя оказалось минимальным. Известно, что решение единственно.

**Входные данные** (текстовый файл):
- Первая строка: N и Z
- Вторая строка: значения весов (в кг) для каждого артефакта
- Третья строка: величина налога для каждого артефакта

**Выходные данные:**
- Порядковые номера выбранных артефактов
- Суммарный вес выбранных артефактов
- Общее налоговое бремя

## Структура программы

Логика программы разбита на отдельные методы с четкими ответственностями:

| Метод | Ответственность |
|-------|------------------|
| `WriteArtifacts()` | Чтение исходных данных из файла |
| `FindMinimalBurden()` | Основной алгоритм поиска оптимального решения (динамическое программирование) |
| `SelectArts()` | Восстановление ответа (какие именно артефакты были выбраны) |

### ✅ Декомпозиция
- Весь код был вынесен из `main()` в отдельные методы
- Каждый метод решает только одну задачу

### ✅ Стиль кодирования (Google C++ Style)
- Исправлены неинформативные названия переменных (`N` → `amount_of_artifacts`, `dp` → `min_tax`, `time` → `burden`)
- Убрано использование `using namespace std;`
- Код приведен к единому стилю форматирования

### ✅ Безопасность и читаемость
- Удалены "магические числа" (999, 9999)
- Введена константа `kInf = INT_MAX / 4` для безопасного представления бесконечности
- Добавлены комментарии `FIX_ME`, указывающие на потенциальные проблемные места

### ✅ Корректность вывода
- Исправлена ошибка, при которой номера артефактов выводились в обратном порядке
- Теперь ответ отображается правильно: от первого артефакта к последнему
